
Java Virtual Machine (JVM) :  자바 언어로 작성된 프로그램을 실행하기 위한 가상 환경으로
자바 바이트 코드를 실제 기계어 코드로 변환하고 실행하는 역할을 담당한다.
JVM은 플랫폼 독립성을 제공하며, 메모리 관리와 가비지 컬렉션을 자동으로 처리합니다.

Java Runtime Environment (JRE) : Java 애플리케이션을 실행하기 위한 런타임 환경으로,
JVM과 자바 클래스 라이브러리 등을 포함하여 필요한 모든 구성 요소를 제공한다.
JRE는 개발자가 아닌 일반 사용자가 Java 프로그램을 실행할 때 필요하다.

Java Development Kit (JDK) : 자바 애플리케이션을 개발하기 위한 도구 모음
자바 컴파일러, 디버거, 런타임 환경 (JRE), 그리고 다양한 개발 도구 및 라이브러리를 포함합니다.

가비지 콜렉터 : 사용하지 않는 (그 어떤 참조자료형의 변수도 참조하지 않는) 객체를 힙 메모리에서 자동으로 삭제시켜주는 프로그램

- 자바는 직접 객체의 메모리를 명시적으로 해제시켜주는 메소드가 없기 때문에 JVM에 내장된 기능중 하나인 가비지 콜렉터를 사용해
자동으로 메모리를 관리한다.
- 가비지 콜렉션은 자바의 핵심적인 기능 중 하나로, 개발자가 메모리 누수와 같은 일반적인 문제들을 방지하는 데 큰 도움을 준다.

플랫폼 독립성 : 프로그램이 특정 하드웨어나 운영 체제에 종속되지 않고 여러 플랫폼에서 동일한 방식으로 작동할 수 있는 성질
